### [2026-04-10 18:54:49 +08:00]

- 解释为什么可以 ping GitHub 但 git push 仍可能失败，结合当前机器的 ICMP/TCP/HTTPS 实测给出结论。
- 本轮实测 ping github.com 4/4 成功，地址为 20.205.243.166，平均约 114ms；Test-NetConnection github.com -Port 443 也返回 TcpTestSucceeded=True；curl -I https://github.com 返回 HTTP/1.1 200 OK。
- 这说明问题不在“完全到不了 GitHub”，而在 Git HTTPS 传输的稳定性：ping 只验证 ICMP 可达，Test-NetConnection 只验证 TCP 443 能建连，curl 首页也只是一次很短的 HTTPS 请求；而 git push 需要更长时间的 TLS/HTTP 会话、鉴权和上传过程，对瞬时丢包、连接重置、代理/中间盒干扰更敏感。
- 结合前面多轮现象，当前机器更像是存在间歇性网络抖动或中间链路重置，所以会出现“能 ping、偶尔能 curl/ls-remote，甚至抓住窗口能 push 一部分仓，但随后又在 github.com:443 上 Could not connect 或 Connection was reset”的情况。
