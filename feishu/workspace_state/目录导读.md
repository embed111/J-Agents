# workspace_state 目录导读

## 作用
`workspace_state/` 用于维护当前工作区的状态恢复、阶段留痕、运行产物说明与执行日志。

## 当前结构
1. `core/`
   1. `startup-checklist.md`：新回合启动与恢复时必须先检查的基线项。
   2. `session-snapshot.md`：当前阶段、最近产出、阻塞点、下一步。
2. `runs/`
   1. 每次本地执行会生成一个独立 run 目录，保存抽取结果、分析结果、草稿和写回结果。

## 更新规则
1. 每完成一个阶段，优先更新 `core/session-snapshot.md`。
2. 若运行链路发生变化，同时更新 `core/startup-checklist.md`。
3. `runs/` 只记录执行事实，不代替核心状态快照。
4. 状态文件必须比对话描述更可信，恢复工作时优先以这里为准。
