# user_profile 目录导读

## 作用
`user_profile/` 只记录当前用户在本工作区的稳定使用偏好与环境假设，帮助后续回合快速恢复，不保存敏感凭据。

## 当前已知偏好与假设
1. 用户希望直接开工，不停留在方案讨论。
2. 用户优先接受浏览器自动化方案，不希望第一轮被 API 集成阻塞。
3. 用户希望复用本机已登录飞书会话，不输入账号密码。
4. 用户要求写回前必须人工确认。
5. 用户要求优先增量更新当前工作区，而不是粗暴覆盖。

## 维护边界
1. 不记录账号密码、Cookie、访问令牌、浏览器 profile 明文路径等敏感信息。
2. 仅记录对后续实现确实有帮助的稳定偏好与环境前提。
