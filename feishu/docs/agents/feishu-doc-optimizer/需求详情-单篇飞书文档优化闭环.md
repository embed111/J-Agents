# 需求详情 - 单篇飞书文档优化闭环

## 1. 主题目标与价值说明
1. 目标：让用户通过一条飞书文档链接，即可完成单篇文档的读取、分析、优化稿生成与确认后写回。
2. 价值：
   1. 避免手工复制飞书正文到外部工具再优化。
   2. 把文档优化沉淀为标准闭环，而不是一次性手工操作。

## 2. 用户画像与使用场景
1. 用户画像：
   1. 企业员工，能访问目标飞书文档。
   2. 更关心“先能用”而非标准 API 集成。
2. 使用场景：
   1. 优化一篇 Wiki 页面结构。
   2. 重写摘要、标题层级、列表表达。
   3. 清理冗余表述、提升阅读路径。

## 3. 用户旅程或关键流程
1. 用户提供一篇飞书文档链接。
2. Agent 复用本机已登录的飞书浏览器会话打开该页面。
3. Agent 抽取标题、结构与正文内容。
4. Agent 输出：
   1. 文档问题诊断。
   2. 优化建议。
   3. 优化稿或重组稿。
5. 用户确认是否写回原文档。
6. 确认后再执行写回。

## 4. 功能需求清单
1. FR-FD-01 单篇链接输入
   1. Agent 必须支持输入单篇飞书 Wiki / Doc 链接。
   2. 链接无法访问时，必须返回可读错误原因。
2. FR-FD-02 正文与结构抽取
   1. Agent 必须能抽取页面标题、段落、列表、标题层级。
   2. 若遇到复杂内容无法稳定抽取，必须明确标记未抽取部分。
3. FR-FD-03 文档问题诊断
   1. Agent 必须输出至少三类诊断：
      1. 结构问题。
      2. 表达问题。
      3. 信息密度或重复问题。
4. FR-FD-04 优化建议与优化稿
   1. Agent 必须同时输出：
      1. 简要问题清单。
      2. 建议调整项。
      3. 可直接使用的优化稿。
5. FR-FD-05 写回确认门禁
   1. 未经用户明确确认，不得直接写回原文档。
   2. 用户确认前默认停留在建议/草稿阶段。
6. FR-FD-06 写回结果反馈
   1. 写回成功后必须返回成功提示与写回范围说明。
   2. 写回失败时必须返回失败原因，不得伪装成功。

## 5. 非功能需求
1. 安全：
   1. 不保存飞书账号密码。
   2. 默认复用用户已登录浏览器会话。
2. 可用性：
   1. 登录态失效时必须能提示用户重新登录，而不是静默失败。
3. 透明性：
   1. 抽取失败、写回失败必须明确提示是“权限问题 / 登录问题 / 页面结构问题”中的哪一类。

## 6. 验收标准
1. AC-FD-01 链接可读
   1. Given 用户输入可访问的飞书文档链接
   2. When Agent 执行读取
   3. Then 能返回页面标题和正文内容
2. AC-FD-02 问题诊断可用
   1. Given Agent 读取到文档正文
   2. When Agent 完成分析
   3. Then 输出结构问题、表达问题、信息密度问题
3. AC-FD-03 优化稿生成
   1. Given Agent 完成文档分析
   2. When Agent 输出结果
   3. Then 同时包含建议和可落地的优化稿
4. AC-FD-04 人工确认门禁
   1. Given 用户尚未确认写回
   2. When Agent 已生成优化稿
   3. Then 不执行写回
5. AC-FD-05 写回反馈
   1. Given 用户明确确认写回
   2. When Agent 执行写回
   3. Then 返回成功或失败结果，不得静默结束

## 7. 边界条件与异常处理
1. 登录态失效：
   1. 终止读取或写回，并提示重新登录。
2. 页面部分内容无法抽取：
   1. 明确标记缺失块，不得假装完整抽取。
3. 无法写回：
   1. 保留优化稿结果，提示用户手工粘贴或稍后重试。

## 8. 依赖项与开放问题
1. 依赖项：
   1. 浏览器自动化运行环境。
   2. 本机已登录飞书会话。
2. 开放问题：
   1. 第一版是否支持表格与折叠块的结构化改写。
   2. 是否需要输出写回前后的 diff 预览。
