# 执行提示词：角色策略详情顺序与评分机制优化（FR-31~FR-37 / AC-36~AC-43）

你在 `../workflow` 工作区执行本任务。

## 1. 目标
按需求文档实现“角色策略详情信息架构重排 + 评分机制可解释化升级”，重点满足以下用户确认点：
1. 首屏优先顺序：`角色/目标` -> `职责边界` -> `门禁得分项`。
2. `门禁与来源` 默认折叠。
3. `AGENTS.md 原文证据` 默认折叠。
4. 新增独立模块：`职责边界`（合并职责与限制，含 `must` / `must_not` / `preconditions`）。
5. 评分机制升级：维度化评分 + 扣分原因 + 证据映射 + 最小修复建议。

## 2. 必读文档（先读后改）
1. `docs/workflow/需求概述.md`（第 13 章）
2. `docs/workflow/需求详情-训练闭环监控系统-Phase0.md`（第 13 章，FR-31~FR-37，AC-36~AC-43）

## 3. 范围约束
1. 本次仅实现“会话前角色策略详情视图”相关改造。
2. 不做无关重构；不改动与本需求无关的训练队列、A/B 切换主流程。
3. 若需新增字段，优先做向后兼容（旧字段保留，不破坏现有接口调用）。

## 4. 实现要求

### 4.1 前端信息架构（`scripts/day3_web_app.js` + 页面模板）
1. 调整确认弹窗展示顺序为：
   1. 角色/目标
   2. 职责边界（must/must_not/preconditions）
   3. 门禁得分项
   4. 门禁与来源（默认折叠）
   5. AGENTS.md 原文证据（默认折叠）
   6. 风险提示
   7. 编辑区（保持现有顺序）
2. 默认展开策略：
   1. 展开：角色/目标、职责边界、风险提示
   2. 折叠：门禁得分项、门禁与来源、AGENTS.md 原文证据
3. 不允许把“门禁与来源”放在首屏第一块。
4. 保留复制导出能力（HTML/截图），但不改变主路径决策顺序。
5. 不允许首屏继续出现“角色/职责/目标 + 限制内容”双模块。
6. `角色/目标`模块仅承载角色与目标；职责约束统一进入`职责边界`模块。

### 4.2 职责边界模块（新增）
1. 从策略提取结果中结构化产出并展示：
   1. `must`（必须项）
   2. `must_not`（禁止项）
   3. `preconditions`（前置条件）
2. 每条职责边界条目需关联证据片段（来自 AGENTS.md）。
3. 若职责边界条目缺失证据或互相冲突，门禁不得 `auto`，至少降为 `confirm`。

### 4.3 评分机制 v2（后端 + 前端展示）
1. 评分维度至少包含：
   1. 完整性
   2. 可执行边界
   3. 一致性
   4. 可追溯性
   5. 风险覆盖度
   6. 可操作性
2. 输出结构必须包含：
   1. 每维度分数
   2. 总分与权重（可配置或可说明）
   3. 扣分原因
   4. 证据映射
   5. 最小修复建议（明确建议修改 AGENTS 哪类段落）
3. “无证据不扣分”：某维度无证据时，不得直接下扣分结论，需标记待人工确认。
4. 增加评分版本字段（如 `score_model=v2`），并写入可追溯日志。

### 4.4 接口与数据兼容
1. 现有 `policy_confirmation` 载荷保持可用。
2. 在不破坏旧字段的前提下新增：
   1. `constraints`（must/must_not/preconditions）
   2. `score_model`
   3. `score_dimensions`（含原因、证据、修复建议）
3. 前端对新老载荷都可渲染（新字段缺失时降级展示）。

7. AC-42：评分版本可追溯。
8. AC-43：职责边界三分类可见，且与门禁联动。
