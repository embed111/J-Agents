# 用户思维模式概述

- 最后更新: 2026-03-11
- 维护目标: 持续记录用户在不同领域的思考与决策习惯，支持后续更精准的需求澄清与交付。

## 使用规则
1. 每轮对话后更新变更日志。
2. 遇到明显模式变化时更新对应领域文件。
3. 仅将“稳定出现”的行为写入长期模式；一次性表达先记为“待验证观察”。
4. 非用户明示结论必须标注“推断/假设”并给出置信度。
5. 仅记录 owner（用户本人）偏好；trainer/外部角色偏好不得写入本目录。
6. 长期偏好文件默认单文件不超过 200 行；超过即触发评审机制。
7. 超限评审后，未保留条目降级到短期偏好池，不直接删除。
8. `thinking-patterns-change-log.md` 属于流水日志，不纳入 200 行上限。

## 置信度说明
- 高: 在多个回合、多个场景重复出现。
- 中: 在单一场景重复出现，尚未跨场景验证。
- 低: 仅出现一次或语义仍有歧义。
