# 用户思维模式 - 全局

- 最后更新: 2026-03-11

## 稳定模式（跨主题）
1. 模式: 倾向先定义角色职责与边界，再进入执行。
   - 置信度: 高
   - 依据: 连续要求先建立并明确 `AGENTS.md` 职责规范。
   - 对协作的影响: 先给清晰框架，再给具体产物更符合预期。
2. 模式: 偏好结构化文档与可维护的知识沉淀。
   - 置信度: 高
   - 依据: 明确要求“概述文件 + 引用多个详情文件”的组织方式。
   - 对协作的影响: 输出应优先使用分层文档而非一次性答案。
3. 模式: 重视长期适配而非单轮回答。
   - 置信度: 中
   - 依据: 主动提出“每次对话都持续维护思维模式文件”。
   - 对协作的影响: 需要保留时间序列记录并跟踪模式变化。
4. 模式: 偏好“可迁移的方法论库”，而非仅依赖即时回答。
   - 置信度: 高
   - 依据: 明确要求收集分析方法并整理对应案例。
   - 对协作的影响: 每次分析需显式标注方法与适用场景。
5. 模式: 强调重启后状态一致性，降低上下文依赖。
   - 置信度: 高
   - 依据: 明确要求“每次重新启动工作区后保持相同工作状态”。
   - 对协作的影响: 需要固定启动读取顺序与会话快照维护机制。
6. 模式: 偏好助手“举一反三”，在完成任务后主动给更优实践方案。
   - 置信度: 高
   - 依据: 明确提出“如果有更好的实践方式也可以提出来”。
   - 对协作的影响: 默认在交付后给 1~3 条高杠杆改进建议并说明取舍。
7. 模式: 倾向由助手主导需求澄清流程，而非用户逐步驱动。
   - 置信度: 高
   - 依据: 用户明确询问“如何让 agent 引导我完善需求分析过程，而不是我引导 agent”。
   - 对协作的影响: 每轮应先由助手给出阶段判断与关键澄清问题，再进入细节讨论。
8. 模式: 长期稳定不退化优先于单次最优输出，并接受分阶段优化目标切换。
   - 置信度: 高
   - 依据: 用户明确确认“想要长期不退化并逐渐迭代变好”，且提出系统初期进化速度占比 70%。
   - 对协作的影响: 方案应采用“早期探索偏重速度、后期收敛偏重稳定”的动态策略，而非固定权重。
9. 模式: 设计与分析偏好以明确方法论约束为前提，尤其重视 `逻辑学 / 第一性原理 / 辩证法`。
   - 置信度: 高
   - 依据: 用户明确要求将该方法论要求写入长期偏好。
   - 对协作的影响: 涉及页面、流程和机制设计时，应先校验论证是否自洽、是否回到问题本质、是否处理了主要矛盾与阶段变化。

## 待验证观察
1. 观察: 在不同领域会接受不同分析深度与表达风格。
   - 置信度: 中
   - 当前处理: 后续按领域分别记录，不提前统一假设。
2. 观察: 对分析方法的探索意愿高，愿意引入新方法而非固定三种。
   - 置信度: 高
   - 当前处理: 持续扩展方法库并配套案例，按任务动态选法。
