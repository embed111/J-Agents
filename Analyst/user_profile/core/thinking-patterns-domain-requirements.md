# 用户思维模式 - 需求分析场景

- 最后更新: 2026-03-02

## 场景特征
1. 关注点: 不满足于“表面需求”，强调挖掘底层动机与根本问题。
2. 交付偏好: 需求结论要可读、清晰，并可被团队直接使用。
3. 过程偏好: 先建立方法论，再落到模板和执行细节。
4. 能力定位（用户明示）: 自身并非专业需求分析人员，需要 agent 提供主动引导而非被动应答。
5. 新增痛点（用户明示）:
   - 过长上下文会降低处理质量（人和 agent 都受影响）。
   - 留痕文件过多会抬高每次启动的读取成本。
6. 新增阶段目标（用户明示）:
   - 训练初期需要训练师能快速创建可用角色，支持业务快速启动。
7. 新增治理偏好（用户明示）:
   - 强调产品边界清晰：`workflow` 作为独立产品需求管理，不并入训练师方案主体。
8. 新增平台规则偏好（用户明示）:
   - agent 由目录契约发现：`C:\work\agents` 下存在 `AGENTS.md` 的目录即有效 agent。
   - 新会话创建时加载 `AGENTS.md`，目录名作为 agent 标识。
   - `AGENTS.md` 视为白名单只读文件，仅在训练师升级流程中更新。
   - 并行会话首版上限固定为 5。
   - 会话创建门禁：新会话第 1 步必须先选择 agent；若无可用 agent，禁止创建会话。
9. 新增体验验收偏好（用户明示）:
   - 对话窗口应与 CLI 保持行为等效（不仅是页面样式相似）。
10. 新增并发交互规则（用户明示）:
   - 并行上限为 5 时，允许多个会话同时生成回复（跨会话并发），不采用全局串行生成模型。
11. 新增执行架构偏好（用户明示）:
   - 前端职责收敛为 API 调用与结果展示；后端负责命令行 agent 编排执行、上下文注入和结果留痕。
   - 对话体验可通过“执行过程回放/增量事件推送”实现，不要求前端直接承载命令行会话本体。
12. 新增权限控制偏好（用户明示）:
   - 接受权限下放，但要求限制在“指定 agent 目录”内，不接受全盘写入。
   - 界面需支持设置并展示当前生效的 agent 路径，作为会话执行权限边界。
13. 新增目录所有权规则偏好（用户明示）:
   - 权限判定优先按 `AGENTS.md` 存在性分层：
     1) 无 `AGENTS.md` 的目录按“开放写入区”处理；
     2) 有 `AGENTS.md` 的目录按“agent 专属写入区”处理，仅当前会话 agent 可写。
   - `C:\\work\\agents` 需支持默认路径配置，并允许在界面可见和调整。
14. 阶段性策略覆盖（用户明示）:
   - 当前阶段暂缓细粒度权限控制，先采用“有 agent 即可执行目录写入”的快速模式。
   - 默认路径当前语义固定为“agent 搜索根路径”，而非强隔离权限根。
15. 写入边界补充（用户明示）:
   - 当前快速模式下，写权限限定为“指定目录 + 子目录”，不外溢到其他目录。
16. 配置语义补充（用户明示）:
   - `agent_search_root` 统一承担双重职责：
     1) agent 列表搜索根路径（发现 `AGENTS.md`）；
     2) 写权限边界根路径（仅允许该路径及子目录写入）。
   - 路径切换为全局敏感操作：切换后必须关闭全部对话会话，防止会话权限上下文漂移。
17. 页面信息架构偏好（用户明示）:
   - 对话页需更接近“微信式会话结构”：会话列表与主聊天窗分离，避免把多类操作堆在同一侧栏。
   - 当前对话模块占比可适度压缩，以给核心会话区域更清晰的视觉与操作重心。
18. 训练工作流模块化偏好（用户明示）:
   - 训练不是按钮动作集合，而是核心工作流：
     1) 新增 agent 对话记录；
     2) 指派分析师并可视化分析过程；
     3) 产出可视化训练计划并允许用户勾选执行。
19. 阶段范围取舍（用户明示）:
   - `workflow` 自升级功能可先移除或下线，待主工作流稳定后再评估是否恢复。
20. 分析扩展评估偏好（用户明示）:
   - 需求分析时默认增加一层“是否需要新增 skill 或 agent 角色”的判断，不再只围绕现有能力做方案。
21. 偏好归属治理偏好（用户明示）:
   - 训练师用于提升分析能力时，训练师偏好不得被记录为用户偏好。
22. 调用链路可视化偏好（用户明示）:
   - 对话系统需支持完整调用链路与提示词按需展开查看，并保持默认折叠。
23. 交互一致性补充偏好（用户明示）:
   - 对话上下文需显式纳入 AGENTS 职责约束；运行态需会话级可见；路径输入需支持 `$root` 类别名。
24. 会话配置作用域偏好（用户明示）:
   - `AGENTS.md` 变更仅影响新会话，历史会话继续使用创建时快照。
25. 路径语义作用域偏好（用户明示）:
   - `$root` 绑定全局 `agent_search_root`，用于跨会话统一路径引用（含其他工作区文件）。
26. 文档一致性治理偏好（用户明示）:
   - `docs/workflow` 在分析工作区与执行工作区需保持同步，避免实现方读取到过期需求。
27. 追溯完整性优先偏好（用户明示）:
   - 已分析且已生成训练计划的对话记录禁止删除，优先保障训练链路可追溯性。
28. CRUD 评审门禁偏好（用户明示）:
   - 每个功能需求必须显式审查增删改查四类能力（需要/可选/禁用 + 原因）。
29. 单点修改受控分发偏好（用户明示）:
   - 需求调整先在当前工作区 `docs/` 落盘，再由用户指派分发到其他工作区。
30. 新增会话策略强约束偏好（用户明示）:
   - 用户明确要求“角色/目标”必须从所选 agent 的 `AGENTS.md` 自动提取，不接受手工输入替代。
   - 用户明确要求策略必须在每一轮对话中严格执行，不接受仅做展示。
31. 新增最小验收偏好（用户明示）:
   - 最小目标采用 `C`：角色/目标不仅影响聊天回复，还必须影响分析与训练决策链路。
32. 新增身份问答偏好（用户明示）:
   - 用户希望“你是谁/职责是什么”类问题使用稳定的身份说明模板（`我是一名...agent，我能...`）。
   - 该模板必须绑定会话冻结策略，不接受脱离 `AGENTS.md` 的通用回答。
33. 新增会话创建门禁偏好（用户明示）:
   - 首次打开主界面时默认不选 agent，先选 agent 再创建会话。
   - 选择 agent 后需等待角色/职责/目标分析完成，期间禁止创建会话。
34. 新增缓存新鲜度偏好（用户明示）:
   - 角色分析可使用缓存，但缓存有效性必须与 `AGENTS.md` 修改时间/版本对齐。
   - 若缓存早于文件变更，必须触发重算，不接受静默复用旧缓存。
35. 新增兜底取舍偏好（用户明示）:
   - 接受“手动输入角色/职责”作为兜底路径，但不接受其替代自动提取主路径。
   - 倾向通过开关受控开放，并要求可追溯审计。
36. 新增可移植性偏好（用户明示）:
   - 希望支持“目录整体复制到新电脑后可直接使用”，降低环境迁移成本与停机时间。
   - 对“同目录同配置可复现”有明确期望。
37. 新增可观测性呈现偏好（用户明示）:
   - 偏好“流程和细节都可见”，当内容过多时采用折叠/展开分层，而不是直接隐藏关键步骤。
   - 目标是支持按需自助探索功能细节与执行链路。
38. 新增目录治理偏好（用户明示）:
   - 文档目录顶层应保持最小化，仅保留统一导读入口，减少启动与巡检成本。
   - 当状态文件变大时，优先采用阈值触发的自动归档机制。

## 决策偏好（当前）
1. 偏好“可维护”的需求管理，而非一次性文档。
   - 置信度: 高
2. 偏好“跨层次”的需求表达（概述层 + 详情层）。
   - 置信度: 高
3. 可能重视“角色自约束”来保证长期质量。
   - 置信度: 中（推断）
4. 偏好“个性化引导”而非“通用默认分析流程”。
   - 置信度: 高

## 对应执行策略
1. 先明确目标、范围、验收标准，再补充功能细节。
2. 每次输出包含“已确认事实 / 推断假设 / 待确认问题”三类信息。
3. 以版本化方式维护需求文档，保留变更理由。
4. 默认先做“表达习惯对齐”再进入正式需求拆解，减少分析偏差。
5. 留痕策略采用“分层存储 + 启动最小读取”：
   - 启动仅读摘要索引与会话快照。
   - 详细记录按需回溯，不在每次启动全量加载。
6. 文档治理优先“职责解耦”：
   - 训练机制文档与产品需求文档分开维护，避免目标混杂导致执行失真。
7. 平台行为治理默认“会话级冻结”：
   - 会话创建时固化 `agents_version/agents_hash`，会话进行中不受后续文件变更影响。
   - 配置更新仅影响新会话，避免长会话行为漂移。
8. 产品迭代治理默认“主链路优先”：
   - 先保证“会话体验 + 训练工作流可观测”闭环，再恢复低优先级运维能力。
9. 方案论证默认加入“能力扩展门禁”：
   - 先判断是否需新增 skill/角色，再决定只做功能调整还是做角色/能力重构。
10. 偏好档案写入默认加入“来源门禁”：
   - 先做来源校验（owner/external），再决定写入 `user_profile` 或外部观察日志。
11. 需求交付默认加入“跨工作区文档一致性校验”：
   - 发布前校验 `docs/workflow` 双目录版本一致（文件集 + 哈希），防止执行偏差。
12. 功能变更评审默认加入“CRUD 完整性门禁”：
   - 无 CRUD 声明或禁用原因不清晰时，默认退回补充。
13. 追溯相关删除策略默认采用“禁止优先”：
   - 一旦已生成训练计划，相关对话删除能力默认禁用，仅允许审计说明。
14. 会话策略治理默认采用“自动提取 + 会话冻结 + 全链路注入”：
   - 入口提取失败时默认 fail-closed，不降级为通用助手口径。
   - 对话与训练链路共用同一份会话策略，不允许分叉。
15. 会话入口治理默认采用“先分析后放行 + 缓存失效重算”：
   - 未选 agent 或策略未就绪时，“创建会话”默认禁用。
   - 命中有效缓存可加速放行，但缓存过期时必须重算后再放行。
16. 迁移治理默认采用“环境自检先行”：
   - 启动前先校验外部依赖（Python/Codex/关键环境变量）和路径可达性。
   - 未通过自检时 fail-closed 并返回可执行修复指引，避免“复制后静默失败”。
17. 交互信息架构默认采用“摘要在前 + 细节下钻”：
   - 主界面展示当前阶段、门禁状态与结果摘要。
   - 每个关键阶段提供可折叠原始证据（提示词、payload、审计记录）。
18. 状态目录治理默认采用“单入口 + 动态归档”：
   - `user_profile/` 与 `workspace_state/` 顶层只保留 `目录导读.md`。
   - 通过阈值巡检脚本触发归档，避免大文件长期膨胀。
