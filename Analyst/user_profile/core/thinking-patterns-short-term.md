# 用户思维模式 - 短期偏好池

- 最后更新: 2026-03-03
- 目的: 存放“已观察但尚未满足长期稳定阈值”或“长期主档超限后被降级”的偏好条目。

## 使用规则
1. 本文件不直接驱动长期默认策略，仅作为候选池。
2. 条目满足“跨 2 个独立话题复现 + 行为模式一致”后，可回升到长期偏好文件。
3. 每次长期偏好超限评审时，优先把阶段性或单主题条目降级到本文件。

## 2026-03-03 超限评审降级（来源: 需求分析域）
### A. 2026-02-28 增量（实践分类）
1. 实践分类偏好：
   1. 将 AI 实践分为：
      1) 基于 agent 构建产品；
      2) 基于产品构建 agent。
2. 失败归因框架：
   1. A 类关键失败：需求澄清不足、流程细节不可见。
   2. B 类关键失败：agent 上下文不足、产品说明不足。
3. 配套执行策略候选：
   1. 案例分析前先做 A/B 类型判定门禁。
   2. 失败复盘采用“共同根因 + 类型特异因子”双层归因。
4. 相关衍生偏好候选：
   1. 每轮分析默认规避 A/B 两类失败。
   2. 关键输入缺失时先澄清，不直接给实现建议。
   3. 输出顺序“先结论后补充”，且显式呈现潜台词与底层需求。
   4. 推断对外展示阈值 `confidence >= medium`。
   5. 阶段取舍上“闭环质量优先于效率”。

### B. 2026-02-28 增量（角色策略详情语义收敛）
1. 语义偏好候选：
   1. `职责` 与 `限制` 合并为 `职责边界`。
   2. “真正职责”由 `角色/目标` 承载，边界模块只表达约束。
2. 验收偏好候选：
   1. `角色设定门禁得分项` 默认折叠。
   2. 验收优先看语义合并是否正确，再看表现层。
3. 对应策略候选：
   1. 角色策略详情默认“双层语义建模”。
   2. 模块合并需求默认加入反向断言（禁止旧模块回流）。

### C. 2026-03-02 增量（工作记录分析模块重设计）
1. 分析输入候选：
   1. 工作记录分析需覆盖对话文本、角色提取信息、兜底状态、执行轨迹。
2. 分析方向候选：
   1. 同时分析角色能力与执行目标/流程。
3. 路由候选：
   1. 能力问题 -> 训练计划。
   2. 流程问题 -> 流程修复。
   3. 符合预期 -> 明确提示，不建训练任务。
4. 执行闭环候选：
   1. 训练执行需有状态流转与结果回写，失败后可复盘。
5. 对应策略候选：
   1. 涉及训练计划生成时先输出四件套：分析对象、分析方向、路由矩阵、执行状态机。

### D. 2026-03-02 增量（需求确认门禁）
1. 节奏偏好候选：
   1. 先确认需求，再开始设计。
   2. 未确认需求时，设计动作延后。
2. 提问与收敛候选：
   1. 单轮提问上限 3 个。
   2. 发现发散风险需主动提示并收敛。
3. 交付与治理候选：
   1. 默认交付“需求概设 + 详细设计”。
   2. 技能沉淀“价值优先”，不为凑数新增。
   3. 偏好提取与需求事实解耦，稳定阈值为跨 2 个独立话题复现。
4. 证据与提示词治理候选：
   1. 验收需可视化证据（截图/录屏/GIF）。
   2. 提示词集中目录管理，新增后仅保留最近 7 轮。
   3. 角色卡视觉语义偏向“人物头像式”而非抽象几何图形。
5. 对应策略候选：
   1. 每轮澄清结构：`0~3 问题 -> 发散风险提示（如有） -> 待确认清单更新`。
   2. 技能建议优先通用能力，不默认面向单工作区特化。
   3. 需求文档默认附 AC 取证要求。

## 回升规则
1. 候选条目满足稳定阈值后，迁移到：
   1. `user_profile/core/thinking-patterns-global.md`
   2. `user_profile/core/thinking-patterns-domain-requirements.md`
   3. `user_profile/core/thinking-patterns-domain-collaboration.md`
2. 每次迁移需在 `user_profile/governance/preference-review-log.md` 记录“迁入理由与证据”。
