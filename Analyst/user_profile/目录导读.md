# user_profile 目录导读

## 1. 顶层规则
1. `user_profile/` 顶层仅保留本文件。
2. 其他文档必须放在子目录：`core/`、`governance/`、`logs/`。
3. 若新增顶层文件，需在同轮迁移并记录到 `workspace_state/logs/state-change-log.md`。

## 2. 子目录导航
1. `core/`
   - `core/thinking-patterns-overview.md`
   - `core/thinking-patterns-global.md`
   - `core/thinking-patterns-domain-requirements.md`
   - `core/thinking-patterns-domain-collaboration.md`
   - `core/thinking-patterns-short-term.md`
2. `governance/`
   - `governance/preference-review-log.md`
   - `governance/preference-review-queue.md`
3. `logs/`
   - `logs/thinking-patterns-change-log.md`
   - `logs/change-log-history-index.md`
   - `logs/change-log-history/`

## 3. 动态维护
1. 偏好主档行数门禁：
   - 命令：`scripts/check-long-term-preference-cap.ps1 -WriteQueue`
   - 规则：长期主档单文件 > 200 行时触发评审。
2. 变更日志归档：
   - 命令：`scripts/maintain-state-health.ps1 -AutoArchive`
   - 规则：`logs/thinking-patterns-change-log.md` 过大时自动归档到 `logs/change-log-history/`。

## 4. 关联入口
1. 状态目录入口：`workspace_state/目录导读.md`
2. 全局规则入口：`AGENTS.md`
