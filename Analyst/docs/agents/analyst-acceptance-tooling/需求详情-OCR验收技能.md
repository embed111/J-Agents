# 需求详情 - OCR 验收技能

## 1. 主题目标与价值说明
1. 目标：为 `Analyst` 提供 OCR 技能，用于从页面截图、局部截图或图片证据中提取可校验文本。
2. 价值：
   1. 解决 DOM 无法稳定读取时的文案核验问题。
   2. 让“视觉文本”也能进入结构化验收流程。
   3. 为截图验收提供可检索、可对比的文字结果。

## 2. 用户画像与使用场景
1. 用户画像：
   1. 需要核验 UI 文字、图片中文字或画布渲染结果的 `Analyst` agent。
2. 使用场景：
   1. 从功能截图中提取标题、副标题、按钮文案。
   2. 检查图表、海报、卡片等非 DOM 文本是否符合需求。
   3. 对设计稿、验收截图与页面截图做文案一致性比对。

## 3. 用户旅程或关键流程
1. `Analyst` 提供图片路径，或由浏览器控制技能先产出截图。
2. OCR 技能读取图片并执行文本识别。
3. 技能输出：
   1. 识别文本。
   2. 识别置信度。
   3. 低置信度标记。
   4. 与预期文本的差异结果。
4. `Analyst` 将 OCR 结果纳入验收结论与证据包。

## 4. 功能需求清单
1. FR-OCR-01 多来源图片输入
   1. 技能必须支持读取本地图片文件。
   2. 技能应支持接收浏览器控制技能产出的截图路径。
2. FR-OCR-02 文本识别输出
   1. 技能必须输出完整识别文本。
   2. 第一版应至少支持中文与英文混合文本识别。
3. FR-OCR-03 置信度与低质量标记
   1. 技能必须输出识别结果的置信度或等效质量指标。
   2. 低置信度结果必须显式标记，不能直接当作确定事实。
4. FR-OCR-04 验收对比
   1. 技能必须支持将 OCR 结果与预期文案进行对比。
   2. 对比结果必须指出：
      1. 完全一致。
      2. 部分不一致。
      3. 无法确认。
5. FR-OCR-05 证据输出
   1. 技能必须返回原图路径。
   2. 若实现允许，技能应支持输出标注图或文本块坐标。
6. FR-OCR-06 信任边界提示
   1. 当 DOM 可直接读取时，OCR 结果不得替代 DOM 结果作为唯一事实源。
   2. 技能必须提醒“OCR 为补充证据，不是最高可信事实源”。

## 5. 非功能需求
1. 准确性：
   1. 对清晰、无遮挡、标准字号的中英文文本，应具备可用识别能力。
2. 透明性：
   1. 对低置信度识别必须明确提示。
3. 可审计：
   1. 每次执行至少保留原图路径、识别文本与差异结果。
4. 可组合性：
   1. 应易于与浏览器控制技能串联，形成截图后 OCR 的闭环。

## 6. 验收标准（Given/When/Then）
1. AC-OCR-01 基础识别可用
   1. Given 一张包含清晰中文按钮文案的截图
   2. When 技能执行 OCR
   3. Then 返回对应文案文本与识别结果
2. AC-OCR-02 差异对比可用
   1. Given 用户提供预期文案
   2. When 技能完成 OCR
   3. Then 返回一致/不一致/无法确认的对比结论
3. AC-OCR-03 低置信度可见
   1. Given 截图模糊或背景复杂
   2. When 技能执行 OCR
   3. Then 明确标记低置信度，不得直接输出确定性通过结论
4. AC-OCR-04 证据可追溯
   1. Given 技能完成一次识别
   2. When 用户查看结果
   3. Then 至少能看到原图路径、识别文本和差异摘要

## 7. 边界条件与异常处理
1. 图片过小、模糊或被遮挡：
   1. 技能返回“无法可靠识别”。
2. 文本与背景对比度过低：
   1. 技能允许输出部分结果，但必须标记低可靠性。
3. 图片实际不含可识别文本：
   1. 技能返回“未识别到文本”，不强行编造内容。

## 8. 依赖项与开放问题
1. 依赖项：
   1. OCR 引擎或可调用识别能力。
   2. 图片读取与结果序列化能力。
2. 开放问题：
   1. 第一版是否要求输出文字块坐标。
   2. 第一版是否要求输出标注图。
   3. 是否需要支持表格文字的结构化识别。
