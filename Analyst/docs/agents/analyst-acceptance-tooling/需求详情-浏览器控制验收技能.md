# 需求详情 - 浏览器控制验收技能

## 1. 主题目标与价值说明
1. 目标：为 `Analyst` 提供一项可受控执行的浏览器控制技能，用于直接观察页面状态并完成功能验收取证。
2. 价值：
   1. 让需求验收不再完全依赖用户口述。
   2. 让 `Analyst` 能独立核对页面文案、显隐状态、跳转结果与截图证据。
   3. 复用 `feishu` 工作区已验证的“本机浏览器会话附着”思路，降低首轮实现风险。

## 2. 用户画像与使用场景
1. 用户画像：
   1. 需要验收 Web 页面功能的 `Analyst` agent。
2. 使用场景：
   1. 打开某个本地或线上页面，确认目标模块是否出现。
   2. 校验按钮、文案、状态标签、列表项是否符合需求文档。
   3. 对关键页面进行截图，作为验收证据回传。

## 3. 用户旅程或关键流程
1. `Analyst` 接收待验收页面地址与验收关注点。
2. 技能附着到本机浏览器会话或启动受控浏览器实例。
3. 技能打开目标页面并等待页面进入稳定状态。
4. 技能执行只读检查：
   1. 读取标题、URL、关键文本。
   2. 检查元素是否可见、可点击、是否存在。
   3. 采集整页或局部截图。
5. 技能输出结构化结果、证据文件路径与失败原因。

## 4. 功能需求清单
1. FR-BC-01 浏览器会话接入
   1. 技能必须支持附着已有浏览器会话。
   2. 第一版可限定为 `CDP` 或本地 profile 复用两种模式之一或两者并存。
   3. 不得要求用户把账号密码输入到技能。
2. FR-BC-02 页面访问与稳定等待
   1. 技能必须支持根据 URL 打开目标页面。
   2. 页面未稳定前不得直接进入判定。
   3. 页面加载失败时必须返回明确错误。
3. FR-BC-03 只读验收检查
   1. 技能必须支持至少以下检查类型：
      1. 页面标题读取。
      2. 当前 URL 读取。
      3. 关键文本存在性检查。
      4. 元素显隐状态检查。
      5. 元素数量或列表项存在性检查。
4. FR-BC-04 截图取证
   1. 技能必须支持整页截图。
   2. 技能应支持目标区域截图。
   3. 每次截图必须返回实际保存路径。
5. FR-BC-05 结果结构化输出
   1. 技能必须输出：
      1. 执行时间。
      2. 页面地址。
      3. 检查项列表。
      4. 每项检查结果。
      5. 证据文件路径。
6. FR-BC-06 高风险动作门禁
   1. 默认仅允许只读操作。
   2. 若未来扩展点击、输入等高风险动作，必须显式要求用户确认。
   3. 未确认前不得执行可能改变业务状态的操作。

## 5. 非功能需求
1. 安全：
   1. 不保存账号密码、Cookie 明文或访问令牌。
   2. 默认不执行写操作。
2. 稳定性：
   1. 页面结构变化时必须失败可见，不得伪造通过。
   2. 登录态失效时必须停止并提示用户处理。
3. 可审计：
   1. 每次执行都必须有日志与证据路径。
4. 可扩展性：
   1. 检查项定义应支持后续扩展为统一验收脚本格式。

## 6. 验收标准（Given/When/Then）
1. AC-BC-01 页面打开成功
   1. Given 用户提供可访问页面地址
   2. When 技能执行页面打开
   3. Then 返回页面标题、当前 URL 和至少一张截图
2. AC-BC-02 文案校验成功
   1. Given 页面包含目标文案
   2. When 技能执行只读检查
   3. Then 返回该文案的存在结果和对应证据
3. AC-BC-03 页面异常可见
   1. Given 登录态失效或页面加载失败
   2. When 技能尝试访问页面
   3. Then 流程终止并返回可读错误，不得输出伪成功结果
4. AC-BC-04 高风险门禁有效
   1. Given 技能被请求执行可能改变页面状态的操作
   2. When 用户未给出显式确认
   3. Then 技能必须拒绝执行该操作
5. AC-BC-05 证据完备
   1. Given 技能完成一次验收检查
   2. When 用户查看结果
   3. Then 至少能看到执行时间、页面地址、检查结果与截图路径

## 7. 边界条件与异常处理
1. 页面跳到登录页或二次验证页：
   1. 技能停止继续检查。
   2. 返回“登录态失效/需人工验证”。
2. DOM 结构变化导致定位失败：
   1. 技能返回“定位失败/页面结构已变更”。
   2. 不得自动改用高风险点击兜底。
3. 页面可见但目标文本在画布或图片中：
   1. 技能应提示改用 OCR 能力补充验证。

## 8. 依赖项与开放问题
1. 依赖项：
   1. 浏览器自动化框架。
   2. 本机浏览器访问能力。
   3. 证据输出目录治理规则。
2. 开放问题：
   1. 第一版是否要求支持 `iframe` 页面检查。
   2. 第一版是否要求支持移动端模拟视口。
   3. 证据输出是单次运行一个目录，还是按任务维度聚合。
