# 执行提示词-工程化重构七轮边界收敛与根目录瘦身-20260304

你现在在 `../workflow` 工作区执行“工程化重构第七轮（边界收敛 + 源码根目录瘦身）”。  
本轮目标不是新增功能，而是修正模块划分问题，让结构真正可维护。

## 1. 先读文件（必做）
1. `docs/workflow/requirements/需求详情-工程化重构与目录治理.md`

## 2. 本轮必须完成（两大目标）
1. 目标 A：边界收敛（去反向耦合）
   1. `server/api/*` 不得再依赖 `workflow_app.workflow_web_server`。
   2. `server/services/*` 不得再依赖 `workflow_app.workflow_web_server`。
   3. `server/bootstrap/web_server_runtime.py` 不得直接依赖“根目录历史单体模块”（如 `training_center_runtime.py`、`workflow_entry_cli.py`、`workflow_history_admin.py`）；改为依赖分层后的模块入口。
2. 目标 B：源码根目录瘦身（`src/workflow_app`）
   1. 根目录不承载业务实现，只允许门面/兼容/说明文件。
   2. 业务实现迁入子目录（如 `apps/`、`runtimes/`、`ops/`、`history/`、`adapters/` 等，命名可调整但必须语义化）。
   3. 若保留兼容门面，必须是薄文件（仅转发）。

## 3. 第七轮硬门禁（任一失败即 `fail_hard_gate`）
1. 依赖门禁：
   1. `rg -n "workflow_app\\.workflow_web_server|from workflow_app\\.workflow_web_server import \\*" src/workflow_app/server/api src/workflow_app/server/services` 结果必须为空。
2. bootstrap 反向依赖门禁：
   1. `server/bootstrap/web_server_runtime.py` 中不得出现对 `...training_center_runtime`、`...workflow_entry_cli`、`...workflow_history_admin` 的直接导入。
3. 根目录瘦身门禁（源码根 `src/workflow_app`）：
   1. 根目录 `.py` 文件数 `<= 4`。
   2. 根目录 `.py` 中除 `workflow_web_server.py` 外，不得有业务实现文件（若存在必须 `<= 40` 行且仅做兼容转发）。
   3. 根目录不允许新增 `> 200` 行源码文件。
4. 文档一致性门禁：
   1. `server/bootstrap/README.md` 必须与实际装配机制一致（manifest，不得写“按文件名排序拼装”）。
   2. `server/README.md` 对路由入口描述必须与代码一致。
5. 回归门禁：
   1. UO + AR 脚本均 `exit code 0`。
