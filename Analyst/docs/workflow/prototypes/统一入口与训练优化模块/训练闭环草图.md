## 2. 这个草图要表达的重点

1. 不再把 `训练师` 和 `队列` 放在页面中心。
2. 页面中心改成：
   - 训练目标是什么
   - 本轮加了什么资料和 skill
   - 三轮执行后平均分多少
   - 应该通过、继续，还是回退
3. `回退` 是显式动作，而且只回退“本轮新增的参考资料和 skill”。
4. `三轮评测` 是主动作，不再是单次执行。

## 3. 与当前原型相比的替换关系

1. 当前 `训练师匹配区` -> 替换为 `参考资料与 Skill 工作集`
2. 当前 `训练队列区` -> 替换为 `训练用例与三轮评测`
3. 当前 `执行结果区` -> 替换为 `评分与决策`
4. 当前 `手动计划表单` -> 保留为 `训练计划卡`，但字段改为面向闭环训练

## 4. 后续若进入 HTML 参考图，需要保留的现有风格约束

1. 保留现有 `训练中心` 一级入口与左侧主导航位置。
2. 保留现有卡片、浅色面板、信息分栏的整体风格。
3. 不做脱离 `workflow` 现网的全新视觉语言。
4. 优先做“信息结构正确”，再做视觉细化。

## 5. 当前已追加的展示约束（相对本草图）

1. 任务状态页与创建任务页都不得超出主窗口视口。
2. 左、中、右三栏主显示框底边必须平齐。
3. 超出部分使用显示框内部滚动条承载，不退化为整页长滚动。
4. 滚动体验应接近客户端固定窗口中的局部滚动区，而不是网页长页面滚动。
