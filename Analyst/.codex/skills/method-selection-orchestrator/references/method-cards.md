# Method Cards

## Purpose
Provide quick selection rules so analysis starts from the right method, not habit.

## 1) First Principles
- Use when: Existing workflow is complex, legacy-heavy, or hard to justify.
- Core question: What is truly required, and what is just convention?
- Avoid when: Execution window is too short for deep redesign.

## 2) Formal Logic
- Use when: Claims are controversial or argument quality is low.
- Core question: Are premises explicit, and does the conclusion follow?
- Avoid when: You only have anecdotes and no stable premises.

## 3) Dialectical Materialism
- Use when: Two priorities conflict and must be staged, not binary-selected.
- Core question: What is the main contradiction now, and when does it shift?
- Avoid when: Problem is static and conflict is weak.

## 4) Systems Thinking
- Use when: Outcomes depend on feedback loops, delays, and interactions.
- Core question: Which loop dominates behavior over time?
- Avoid when: Scope is local and causal chain is short.

## 5) Bayesian Updating
- Use when: Decisions must be revised as new evidence arrives.
- Core question: What is the prior belief and how does new evidence update it?
- Avoid when: No measurable signal can be collected.

## 6) Decision Analysis (Expected Value)
- Use when: You need explicit tradeoff between payoff and probability.
- Core question: Which option has the highest risk-adjusted expected value?
- Avoid when: Payoffs are not comparable at all.

## 7) Premortem
- Use when: Team is optimistic and failure modes are underexplored.
- Core question: If this fails in 6 months, what likely caused it?
- Avoid when: Team has not aligned on basic objective yet.

## 8) Second-Order Effects
- Use when: A local optimization can create delayed downstream damage.
- Core question: What happens after the immediate effect?
- Avoid when: Decision impact horizon is intentionally very short.
