# 方法: 逻辑学分析

- 最后更新: 2026-02-24

## 适用问题
1. 需求结论争议大，论证链条不完整。
2. 会议中出现“直觉结论”但缺少前提。
3. 需要检验是否存在逻辑谬误。

## 核心步骤
1. 明确命题: 将讨论对象写成可判断真假的命题。
2. 列出前提: 区分事实前提、价值前提、约束前提。
3. 构建推理链: 明确每一步是演绎、归纳或类比。
4. 检查有效性: 验证是否存在前提缺失、偷换概念、因果倒置等问题。
5. 形成可辩护结论: 标注结论成立条件与适用范围。

## 输出格式
1. 核心命题
2. 前提清单
3. 推理路径
4. 漏洞与修正
5. 条件化结论

## 常见误区
1. 结论正确但论证无效，导致不可复用。
2. 用相关性替代因果性。
3. 用单一案例做普遍结论。
