# 用户思维模式概述

- 最后更新: 2026-02-24
- 维护目标: 持续记录用户在不同领域的思考与决策习惯，支持后续更精准的需求澄清与交付。

## 使用规则
1. 每轮对话后更新变更日志。
2. 遇到明显模式变化时更新对应领域文件。
3. 仅将“稳定出现”的行为写入长期模式；一次性表达先记为“待验证观察”。
4. 非用户明示结论必须标注“推断/假设”并给出置信度。

## 置信度说明
- 高: 在多个回合、多个场景重复出现。
- 中: 在单一场景重复出现，尚未跨场景验证。
- 低: 仅出现一次或语义仍有歧义。
